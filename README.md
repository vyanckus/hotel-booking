# Hotel Booking System

Backend-сервис для бронирования отелей с возможностью управления контентом через административную панель.

## Технологии

- **Java 17**
- **Spring Boot 3.5.11**
- **Spring MVC** - REST API
- **Spring Data JPA** - работа с PostgreSQL
- **Spring Security** - аутентификация и авторизация
- **Spring Data MongoDB** - хранение статистики
- **Apache Kafka** - обработка событий
- **MapStruct** - маппинг сущностей
- **OpenCSV** - экспорт статистики
- **OpenAPI (Swagger)** - документация API
- **PostgreSQL** - основная БД
- **MongoDB** - БД для статистики
- **Docker Compose** - контейнеризация

## Запуск проекта

### Требования
- Docker и Docker Compose
- Java 17
- Gradle

### Шаги для запуска

1. **Клонировать репозиторий**
```bash
git clone <url-репозитория>
cd hotel-booking
```

2. **Запустить инфраструктуру (PostgreSQL, MongoDB, Kafka, ZooKeeper)**


```bash
docker-compose up -d
```

3.  **Собрать и запустить приложение**

```bash
./gradlew bootRun
```

4.  **Доступ к Swagger UI**


```text
http://localhost:8080/swagger-ui.html
```

## API Endpoints

### Пользователи (`/api/users`)

*   `POST /api/users` \- регистрация пользователя (доступно без авторизации)

*   `POST /api/users/with-role?role=ROLE_ADMIN` \- создание пользователя с ролью (доступно без авторизации)

*   `GET /api/users` \- список пользователей (только ADMIN)

*   `GET /api/users/{id}` \- информация о пользователе

*   `PUT /api/users/{id}` \- обновление пользователя

*   `DELETE /api/users/{id}` \- удаление пользователя


### Отели (`/api/hotels`)

*   `GET /api/hotels` \- список отелей (пагинация)

*   `GET /api/hotels/filter` \- фильтрация отелей (ID, название, город, адрес, расстояние)

*   `GET /api/hotels/{id}` \- информация об отеле

*   `POST /api/hotels` \- создание отеля (только ADMIN)

*   `PUT /api/hotels/{id}` \- обновление отеля (только ADMIN)

*   `DELETE /api/hotels/{id}` \- удаление отеля (только ADMIN)

*   `POST /api/hotels/{id}/rate?rating=5` \- оценить отель (1-5)


### Комнаты (`/api/rooms`)

*   `GET /api/rooms` \- список комнат (пагинация)

*   `GET /api/rooms/filter` \- фильтрация комнат (цена, количество гостей, даты, отель)

*   `GET /api/rooms/{id}` \- информация о комнате

*   `POST /api/rooms` \- создание комнаты (только ADMIN)

*   `PUT /api/rooms/{id}` \- обновление комнаты (только ADMIN)

*   `DELETE /api/rooms/{id}` \- удаление комнаты (только ADMIN)


### Бронирования (`/api/bookings`)

*   `POST /api/bookings` \- забронировать комнату

*   `GET /api/bookings` \- список броней (пагинация, только ADMIN)

*   `GET /api/bookings/{id}` \- информация о брони

*   `DELETE /api/bookings/{id}` \- отмена брони (только ADMIN)


### Статистика (`/api/statistics`)

*   `GET /api/statistics/export/csv` \- выгрузка статистики в CSV (только ADMIN)


## Модель данных

*   **User** \- пользователь (username, email, password, роль)

*   **Hotel** \- отель (название, заголовок, город, адрес, расстояние от центра, рейтинг)

*   **Room** \- комната (название, описание, номер, цена, вместимость, отель)

*   **Booking** \- бронирование (даты заезда/выезда, комната, пользователь)

*   **Statistics** \- статистика (события регистрации и бронирования, MongoDB)


## Безопасность

*   Basic Auth

*   Регистрация доступна без авторизации

*   CRUD для отелей и комнат - только ADMIN

*   Просмотр всех броней \- только ADMIN

*   Остальные методы \- для авторизованных пользователей


## Фильтрация

### Отели

*   по ID

*   по названию

*   по заголовку

*   по городу

*   по адресу

*   по расстоянию от центра


### Комнаты

*   по ID

*   по названию

*   по цене (min/max)

*   по количеству гостей

*   по отелю

*   по доступности на даты (проверка броней)


## Статистика (Kafka + MongoDB)

События:

*   **Регистрация пользователя** \- сохраняется ID пользователя

*   **Бронирование комнаты** \- сохраняется ID пользователя, даты заезда/выезда


Данные хранятся в MongoDB, доступен экспорт в CSV для администратора.
